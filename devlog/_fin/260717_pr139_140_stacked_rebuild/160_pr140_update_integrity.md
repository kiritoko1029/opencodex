- A-audit corrections (2026-07-17, claude-fable-5 reviewer — 4 blockers folded):
  - **Gate-before-stop is BINDING.** The gate's principal operational value is aborting BEFORE `ocx stop` unloads the running proxy/service. `runUpdate` must call the gate before the stop block (as the source does), and a test/assertion order must pin it (gate failure ⇒ stop never invoked).
  - **Query-failure ≠ malformed-metadata.** `npm view dist.integrity` nonzero-exit/timeout with a resolved version is a TRANSIENT registry failure: skip the gate with a warning (same lane as null version), do NOT abort. Hard-abort is reserved for a SUCCESSFUL query returning missing/malformed SRI (anomalous metadata = fail closed). Distinguish via spawn status: status!==0 → skip-warn; status===0 with bad output → abort. Test both lanes.
  - **Multi-SRI tolerance.** `dist.integrity` can be space-separated multi-hash (and npm view output may carry quotes/newlines). Parse: strip quotes/trim, split on whitespace, PASS if ANY token matches `^sha512-[A-Za-z0-9+/=]+$`. Test a multi-token fixture.
  - **Honest labeling.** The gate is a pre-flight sanity + fail-closed anomaly check (installers independently verify tarballs against the same registry metadata); it is NOT independent tamper-proofing. Code comments and log lines must say "pre-flight integrity metadata check", not claim tamper defense. GUI npm-path caveat: the Node launcher self-update may still resolve the tag at its own time — note the residual divergence window in a comment rather than claiming full immutability there.
