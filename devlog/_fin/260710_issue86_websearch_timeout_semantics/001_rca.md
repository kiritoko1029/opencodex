# 001 — Root-cause and contract evidence

## Incident signature

Issue #86 reports `lm-studio/qwen3.6-35b-a3b-ud-mlx`, HTTP 504 after 200019 ms, and
`Provider timeout after 200000ms during web-search`. `usageStatus` is unreported and the response is
`non_stream`.

## Causal chain

1. `src/server/responses.ts` routes non-passthrough requests containing hosted web search into
   `runWithWebSearch()` and supplies `config.connectTimeoutMs ?? 200_000`.
2. `src/web-search/loop.ts` rewrites the routed iteration to `stream:false`.
3. LM Studio's OpenAI-compatible chat endpoint therefore withholds a complete JSON response until
   generation ends; its normal token progress is not observable by OpenCodex.
4. The loop creates one `signalWithTimeout(connectTimeoutMs)` and keeps it active through both
   fetch and `adapter.parseResponse()`.
5. At 200 s, the loop maps the abort to the exact issue error. The first iteration is still eager,
   so this becomes non-streaming JSON 504 and `runWebSearch()` has never been called.

This conflicts with `src/types.ts`, where `connectTimeoutMs` is explicitly the DNS/TCP/TLS/response
header deadline, and with the normal routed path, which clears that boundary when fetch returns and
then consumes a stream under stall protection.

## History boundary

- Original Pattern-B web-search loop (`a4053df9`) chose non-stream generation to inspect whole-turn
  tool output.
- `4dccf7e0` fixed forced-answer, mixed real-tool, and total search-budget behavior; these depend on
  whole-turn buffering, not on a non-stream upstream wire.
- `8e503661` introduced eager first-iteration draining to preserve pre-SSE failures while search
  cells became live.
- `852ab046` added the 200 s iteration timeout and 429 rotation.
- `c758f60c` widened the bridge watchdog around the monolithic non-stream unit and documented
  internal streaming as future work.
- `2c149fa3` preserved signed Anthropic thinking before replayed tool use.

The fix can retain whole-turn classification and signed-thinking order while changing only the
upstream wire/parser mode and the point at which downstream SSE is committed.

## Secondary transport hazard

Google and Kiro custom retry helpers create `AbortSignal.timeout(timeoutMs)` for fetch attempts.
That signal remains attached to a successful response body after headers, so switching the loop to
streaming without clearing these timers would reproduce the same total-duration cutoff for those
adapters. Their attempt timers must therefore become header-scoped as part of the same contract.

## Safety invariants

- No synthetic search executes from an errored or protocol-ambiguous buffered iteration.
- Initial response status/429 decisions happen before downstream SSE headers.
- Raw bytes, not only semantic adapter events, count as generation progress.
- Progress keeps an arbitrarily long healthy generation alive; continuous silence remains finite.
- Parent abort and inactivity timeout retain distinct reasons.
- Every reader, timeout, abort listener, and background parser promise settles or is safely caught.
- First and later failure surfaces are based on the actual commit phase, not iteration number.
